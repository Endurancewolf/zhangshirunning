| 项目 | 选型 | 原因 |

| 运行环境 | 浏览器（Chrome / Edge / Firefox） | 无需安装，跨平台 |

## 2. 项目结构

单文件架构：`index.html`

```
index.html
├── <style>          — CSS 样式（居中布局、背景、字体）
├── <canvas>         — 游戏画布元素
└── <script>         — 全部游戏逻辑（见下文模块划分）
```

## 5. 碰撞检测方案

### 方案：AABB 矩形碰撞（Axis-Aligned Bounding Box）

每个游戏对象都有一个碰撞矩形 `{ x, y, width, height }`，两矩形重叠即判定碰撞。

```
碰撞条件：
  A.x < B.x + B.width  &&
  A.x + A.width > B.x  &&
  A.y < B.y + B.height &&
  A.y + A.height > B.y
```

### 碰撞宽容度
- 实际碰撞矩形比显示范围缩小 10-15%，让玩家感觉判定公平
- 跳跃状态角色碰撞区域缩小为上半身
- 低头状态角色碰撞区域缩小为下半身

## 6. 数据持久化

| 存储内容 | Key | 格式 |
|----------|-----|------|
| 历史最高分 | `parkour_highscore` | 数字字符串，如 `"12500"` |

- 使用 `localStorage.setItem()` / `getItem()`
- 读取时做容错处理（首次游玩无记录时返回 0）

---

## 7. 性能注意事项

- 使用 `requestAnimationFrame` 而非 `setInterval`
- 超出画面左侧的障碍物/道具及时从数组中清除（防止内存泄漏）
- 背景复用无限滚动模式，不创建过多对象
- 道具和障碍物使用数组 `filter` 清理，控制数组长度

| 参数 | 值 | 单位 |

| 画布大小 | 900 × 500 | px |
| 基础滚动速度 | 4 | px/帧 |

| 药丸降压量 | 30% | 百分比 |
| 跳跃最大高度 | 120 | px（约 2 倍角色身高） |
| 跳跃持续时间 | ~40 帧 | 帧 |
| 低头持续时间 | 30 帧（0.5 秒） | 帧 |
| 障碍物最小间隔 | 90 帧（1.5 秒） | 帧 |
| 障碍物最小间隔（后期） | 45 帧（0.75 秒） | 帧 |
